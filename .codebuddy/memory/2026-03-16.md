## weibo-cli 搜索功能修复

### 问题
- `weibo search` 命令使用移动端 API（m.weibo.cn），需要移动端 Cookie，PC 端 Cookie 返回 -100 错误
- `weibo home` 命令返回 500 错误（friendstimeline API 需要有效登录态）

### 解决方案
修改了 weibo-cli 源码（`~/.local/share/uv/tools/kabi-weibo-cli/lib/python3.13/site-packages/weibo_cli/`）：

1. **constants.py**：添加 `PC_SEARCH_URL = "https://s.weibo.com/weibo"` 常量
2. **client.py**：
   - 添加 `import re` 和 `from html import unescape`，导入 `PC_SEARCH_URL`
   - 新增 `_strip_tags()` 静态方法
   - 新增 `_parse_search_html()` 方法，解析 s.weibo.com HTML 搜索结果
   - 重写 `search_weibo()` 方法：优先用 PC 端 s.weibo.com，失败回退移动端 API
3. **commands/search.py**：修改 `_render()` 支持新的 `{statuses: [...]}` 返回格式

### HTML 解析细节
- s.weibo.com 返回 HTML（非 JSON），需要正则解析
- `<div class="from"  >` （注意双空格）包含时间和来源
- 转发数在 `woo-font--retweet` 图标后
- 评论数在 `woo-font--comment` 图标后
- 赞数在 `<span class="woo-like-count">` 中
