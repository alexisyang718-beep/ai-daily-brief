# 选题偏好配置

> 本文件定义用户的新闻选题偏好，用于指导AI筛选新闻
> 对应工作流：02-selection（选题报告）

---

## 一、模块优先级（选题时参考）

| 优先级 | 模块 | 说明 |
|:------:|------|------|
| ⭐⭐⭐ P0 | AI模型与产品 | 核心关注，战略调整必选题 |
| ⭐⭐⭐ P0 | 芯片与算力 | 高度关注，产能/价格/供应链 |
| ⭐⭐⭐ P0 | 手机与消费电子 | 全平台均衡关注（苹果/三星/Google/华为/小米） |
| ⭐⭐☆ P1 | 游戏行业 | 专业关注，重视平台商业策略 |
| ⭐⭐☆ P1 | 科技行业动态 | 大厂财报、战略调整 |
| ⭐⭐☆ P1 | 政策与监管 | 地缘政治、合规影响 |

## 二、内容偏好（选题时筛选依据）

### 2.1 强偏好（必选题）

| 类别 | 具体特征 | 例子 |
|------|----------|------|
| **AI模型战略调整** | 大厂产品方向变化、商业策略转向 | OpenAI关闭Sora、Pre-IPO文件泄露 |
| **AI Agent落地** | 智能体产品发布、功能更新 | Claude Code Auto Mode |
| **芯片产能/良率** | 晶圆厂产能、良率数据 | 台积电亚利桑那厂良率追平台湾 |
| **GPU新品/订单** | NVIDIA新品、订单动态 | B300产能爬坡、GB200 |
| **大厂财报数据** | 营收、增速、市场份额 | 字节1500亿美元、OpenAI 9亿用户 |
| **全平台手机动态** | 苹果/三星/Google/华为/小米 | WWDC、Galaxy AI、鸿蒙生态 |
| **游戏平台商业** | 促销、定价、订阅策略 | Xbox春季促销、PSN季度促销 |
| **游戏硬件销量** | 主机销量、生产调整 | Switch 2销量、减产30% |

### 2.2 弱偏好（易淘汰）

| 类别 | 淘汰原因 | 例子 |
|------|----------|------|
| **纯娱乐内容** | 缺乏产业/商业洞察 | 游戏世界地图测试、插画版 |
| **重复报道** | 同一事件多角度 | Cursor套壳Kimi（重复版本） |
| **过于前瞻/概念** | 缺乏实质进展 | 过于技术细节的未发布产品 |
| **边缘创新** | 影响范围有限 | 小众开源项目 |
| **超48小时旧闻** | 时效性不足 | 任何超过48小时的新闻 |

## 三、信源优先级（选题时排序依据）

| 优先级 | 信源类型 | 权重 |
|:------:|----------|:----:|
| ⭐⭐⭐ | **X官方账号** (@OpenAI, @GoogleAI, @NVIDIA等) | 最高 |
| ⭐⭐⭐ | **彭博社/Reuters/财新** | 最高 |
| ⭐⭐⭐ | **9to5Mac/AppleInsider/9to5Google** | 高 |
| ⭐⭐⭐ | **Samsung Newsroom/SamMobile** | 高 |
| ⭐⭐☆ | **微博科技账号** (机器之心/量子位/竞核) | 中 |
| ⭐⭐☆ | **TechCrunch/The Verge** | 中 |
| ⭐☆☆ | **RSS技术博客** | 低 |

**基本规则**：
- 同一新闻优先选高优先级信源
- 官方一手源 ≥ 5条/期
- 同一二手源整期引用 ≤ 2次

**⚠️ 国外信源优先规则（必须严格执行）**：

| 新闻主体 | 信源要求 | 示例 |
|----------|----------|------|
| **国外公司**（OpenAI、Apple、NVIDIA、Sony等） | **必须**用英文官方/X/媒体来源 | 用 CNBC/TechCrunch/9to5Mac，不用量子位 |
| **国内公司**（华为、字节、阿里、腾讯、智谱等） | 可用国内媒体（量子位/机器之心等） | 量子位报道智谱AI年报 ✅ |
| **全球性事件**（芯片价格、行业数据等） | 优先英文分析媒体（Tom's Hardware等） | 不优先选微博账号转述 |

**X 缺失时的处理**：
- 若当日 X 数据为 0 条，须在选题报告开头注明：`⚠️ 本期X数据缺失，AI模块已优先使用英文官方/媒体来源`
- AI模块每条来源必须是英文RSS（官方博客、CNBC、TechCrunch等），不得用量子位/新智元替代国外公司新闻

## 四、时效性要求

| 类型 | 时效要求 | 容忍度 |
|------|----------|--------|
| AI模型发布/更新 | 严格24小时 | ❌ 超24h不选 |
| 财报/战略新闻 | 严格24小时 | ❌ 超24h不选 |
| 芯片产能/价格 | 24-48小时 | ⚠️ 超48h不选 |
| 游戏促销/销量 | 48小时内 | ⚠️ 超48h不选 |
| 政策/监管 | 48小时内 | ⚠️ 超48h不选 |

---

## 五、数量控制

- 每模块精选 **4-6条**
- 总条目控制在 **25-30条**
- 每模块前5条标记 ✅ 为推荐入选

---

## 六、品牌平衡（手机与消费电子模块）

**原则**：全平台均衡关注，不偏向单一品牌

| 品牌 | 关注内容 |
|------|----------|
| Apple | iOS更新、硬件战略、生态布局 |
| Samsung | Galaxy系列、折叠屏、AI功能 |
| Google | Android更新、Pixel、AI原生功能 |
| 华为 | 鸿蒙生态、麒麟芯片、全场景 |
| 小米 | 手机AI、人车家生态、澎湃芯片 |
| OPPO/vivo/一加 | 新品、技术创新、IP联名 |

**选题检查**：
1. 每篇日报手机模块至少覆盖 **3个不同品牌**
2. 同一品牌连续两天头条需有 **差异化角度**
3. 国产厂商与国际品牌比例保持 **4:6 或 5:5**

---

## 七、游戏行业专业关注要点

**高优先级**：
- ✅ 平台商业动态（促销、定价、订阅）
- ✅ 硬件销量数据（主机销量、生产调整）
- ✅ 行业重大事件（裁员、收购、战略调整）
- ✅ 技术争议（AI素材、玩家反弹）
- ✅ AI与游戏的交叉

**低优先级**：
- ❌ 纯娱乐/文化内容（世界地图测试、插画版）
- ❌ 缺乏产业洞察的内容
- ❌ **体育赛事收视率**（腰旗橄榄球、超级橄榄球等，属于体育新闻而非游戏）

---

## 八、模块分类规则（严格边界）

**禁止跨模块重复**：同一新闻只能出现在一个模块

| 新闻内容 | 正确模块 | 错误归类 |
|----------|----------|----------|
| iPad/MacBook/Watch等消费电子 | **手机与消费电子** | AI模型与产品 |
| 华为FTTR/企业网络/基站 | **科技行业动态** | 手机与消费电子 |
| 股市/科技股/大盘 | **科技行业动态** | 其他模块 |
| 体育赛事/收视率 | **不选** | 游戏行业/科技行业动态 |
| 医药/生物科技（非AI制药） | **不选** | 科技行业动态 |

**分类优先级**：
1. 先判断是否为**AI模型与产品**（AI功能/模型更新）
2. 再判断是否为**手机与消费电子**（消费级硬件产品）
3. 再判断是否为**芯片与算力**（芯片技术/产能）
4. 再判断是否为**游戏行业**（游戏平台/硬件/商业）
5. 最后归入**科技行业动态**（其他科技商业新闻）

## 九、排除清单（不关心的内容）

| 类别 | 例子 | 原因 |
|------|------|------|
| 医药健康 | 诺和诺德、减肥药、制药公司财报 | 非科技关注范围 |
| 传统体育 | 超级橄榄球、腰旗橄榄球、足球赛事 | 与科技产业无关 |
| 能源化工 | 石油公司、传统能源债券 | 非新能源科技 |
| 食品饮料 | 消费品公司 | 非关注范围 |

## 十、AI执行检查清单（选题时）

- [ ] 已读取 `news-preferences.md`（本文件）
- [ ] 按6大模块分类，每模块4-6条
- [ ] **检查模块边界**：确保新闻归类正确，不跨模块重复
- [ ] **排除不感兴趣内容**：医药、传统体育、能源化工
- [ ] 前5条标记 ✅ 为推荐入选
- [ ] 检查时效性（主体24h内，补充48h内）
- [ ] **检查信源优先级**：国外公司新闻用英文来源，国内公司才用国内媒体
- [ ] **每条选题含真实文章URL**：从 collect/output 中复制，不用首页URL
- [ ] 检查品牌平衡（手机模块至少3个品牌）
- [ ] 检查游戏内容是否专业（非纯娱乐/非体育赛事）
- [ ] X数据缺失时，在报告开头注明并确保AI模块全部为英文来源
- [ ] 输出到 `selection/selection_YYYYMMDD.md`
- [ ] **等待人工确认后再进入撰写阶段**
