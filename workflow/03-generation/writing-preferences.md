# 撰写偏好配置

> 本文件定义用户对日报撰写的偏好，包括洞察深度、字数、格式等
> 对应工作流：03-generation（撰写日报）

---

## 一、日报模板说明

**模板位置**：`../template.html`

**必须读取**：撰写前必须先读取 `template.html`，了解：
- HTML结构骨架
- CSS样式类名
- 占位符格式（见模板末尾注释）
- 每条新闻的HTML结构示例

**禁止**：自行创建HTML结构，必须基于模板填充。

---

## 二、字数要求

### 2.1 整体控制

| 部分 | 字数范围 | 说明 |
|------|----------|------|
| 单条新闻正文 | 50-100字 | 核心事实，2-3句话 |
| 单条深度洞察 | ≤300字符 | 基于全文+权威观点 |
| 单条标题 | ≤30字 | 简洁有力 |
| 整期日报 | 8000-15000字 | 25-30条新闻 |

### 2.2 洞察字数分配

| 新闻重要性 | 洞察字数 | 要求 |
|:----------:|:--------:|------|
| 头条新闻 | 250-300字符 | 深度分析，多方观点 |
| 普通新闻 | 150-200字符 | 核心洞察，数据支撑 |
| 短讯 | 50-100字符 | 一句话点评 |

---

## 三、内容深度要求

### 3.1 洞察维度（按优先级）

| 优先级 | 维度 | 必须包含 |
|:------:|------|----------|
| ⭐⭐⭐ | **供应链分析** | 产能、良率、价格动态、地缘政治影响 |
| ⭐⭐⭐ | **商业战略解读** | 大厂战略意图、竞争格局、ROI分析 |
| ⭐⭐⭐ | **数据支撑** | 具体数字（营收、增速、市场份额） |
| ⭐⭐☆ | **竞争格局** | 与竞品的对比、市场地位 |
| ⭐⭐☆ | **前瞻预测** | 趋势判断（2-5年） |
| ⭐☆☆ | **技术细节** | 选择性，非必须 |

### 3.2 洞察撰写规则

1. **必须基于全文**：通过Jina Reader/tavily_extract抓取原文
2. **必须有观点支撑**：通过XClaw/Grok搜索权威观点
3. **禁止仅凭搜索摘要写洞察**
4. **禁止空洞描述**：每个洞察必须有数据或事实支撑

---

## 四、信源权威性要求

### 4.1 信源标注

每条新闻在 `item-source` 中只写**来源名称**（如 `9to5Mac`、`CNBC`、`量子位`），**不加类型标签**（一手官方/一手媒体/二手编译等标签用户不需要，禁止标注）。

### 4.2 信源数量控制

- 官方一手源 ≥ 5条/期
- 禁止用新浪/搜狐/腾讯科技作为主要信源
- 同一来源整期引用 ≤ 2次（国内聚合媒体）

---

## 五、原链接要求

### 5.1 链接来源（直接使用选题报告中的URL）

选题报告 `selection/selection_YYYYMMDD.md` 中每条新闻已包含 `**URL**: https://...` 字段，**直接使用该URL**，无需另查JSON文件。

| 情况 | 处理方式 |
|------|----------|
| 选题报告有具体文章URL | 直接使用，填入 `<a href="URL">` |
| URL 为空或是首页根域名 | WebSearch 搜索文章标题补充真实URL |

### 5.2 链接规则

- ✅ 所有新闻必须有**具体文章的可点击链接**（不能是首页）
- ✅ RSS来源用文章链接，微博来源用微博帖子链接，X来源用推文链接
- ❌ 禁止使用信源首页URL（如 `https://9to5mac.com/`）
- ❌ 禁止编造或省略链接
- ❌ 禁止只写来源名称不写链接

---

## 六、格式规范

### 6.1 每条新闻结构

```html
<div class="news-item">
  <h3>标题（≤30字）</h3>
  <span class="tag">信源标签</span>
  <p class="summary">正文（50-100字，核心事实）</p>
  <div class="data-tags">
    <span>数据标签1</span>
    <span>数据标签2</span>
  </div>
  <a href="原始链接" class="source">来源名称</a>
  <div class="insight">
    <strong>💡 深度洞察</strong>
    <p>洞察内容（≤300字符）</p>
  </div>
</div>
```

### 6.2 数据标签要求

每条新闻必须提取1-3个关键数据标签：

| 类型 | 示例 |
|------|------|
| 金额 | $1500亿营收、涨价51% |
| 数量 | 9亿用户、裁员1000人 |
| 百分比 | 增长30%、市场份额25% |
| 时间 | 2026年发布、Q2上市 |

## 七、6大板块顺序

日报板块必须按以下顺序排列：

1. 🤖 **AI模型与产品**
2. 📱 **手机与消费电子**
3. 🔧 **芯片与算力**
4. 🎮 **游戏行业**
5. 🏢 **科技行业动态**
6. 📜 **政策与监管**

- [ ] 字数：正文50-100字，洞察≤300字符
- [ ] 信源：官方一手源 ≥ 5条，同一国内来源引用 ≤ 2次
- [ ] 深度：至少3-5篇全文抓取
- [ ] 洞察：基于全文+权威观点，非仅凭摘要
- [ ] 数据：每条新闻有1-3个数据标签
- [ ] **链接：每条新闻使用选题报告中的具体文章URL，无首页URL**
- [ ] **信源标注：只写来源名称，不加"一手官方/一手媒体/二手编译"标签**

### 8.3 格式检查

- [ ] 基于 `template.html` 填充（非自行创建）
- [ ] 6大板块顺序正确
- [ ] 信源只写名称，无类型标签
- [ ] 数据标签完整
- [ ] 深度洞察 ≤300字符

### 8.4 输出

- [ ] 文件保存到 `brief/YYYY-MM-DD.html`
- [ ] 文件大小正常（30KB-100KB）
